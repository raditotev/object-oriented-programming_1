2. HTML

What is HTML?

HTML documents are the heart of the web. HTML stands for HyperText markup language. HTML is made up of text content (what you see), markup(what that content looks like or how it's arranged), references to other documents (images, videos etc.) and links to other pages.

Tags

HTML markup is made up of things called opening and closing tags. The whole thing might be referred as an element. There are two types of elements: inline and block elements. Inline element is in line with the text (e.g. span), whereas block element forms an imaginary block around the content (e.g. div). There are also void elements such as: br, img and link.

Marking Links

There is a special type of tags with attributes. These are tags withholding attributes inside. Examples of tags with attributes are img, a, link etc. Creating links to other content is done using the src (for images) or href attributes.

Structure

The basic structure of an HTML document consist a head element containing title element and body, all wrapped in html element. As the HTML page grows there will be more elements within element or so called tree like structure.

3. Creating Structured Documents

First Step

Building a web page can be compared with the building of a house. We first start building the foundations, the walls and we finish the basic structure with the roof. This is just like the HTML. After we finish with the basic structure we start work on the external and internal decoration of the house. Painting the walls, furnishing the place and hanging paintings. This is the role of CSS in our document. With CSS we can set different appearance for every single element in our HTML document. Finally we equip our house with some gadgets. Some might might be as simple as automatic garage door some might be as complex as home automation system. This is the place for Javascript in our web page.

HTML-CSS-DOM

HTML is the language of the web. It has syntax and its rules. The basic word in HTML is tag. The browser turns HTML tags into elements that form a tree. It knows how to do that because of the DOM.

CSS - Cascading Style Sheets - style sheet language used for describing the look and formatting of a document written in a markup language.

DOM - Document Object Model - a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree.

Boxes, Grids and Rules

Everything on a web page is a box. By thinking in terms of boxes front end developers can easily rearrange things if they decide to give the users different experience. There is also a grid in the background that serves to make sure that everything is properly aligned.

Boxes are made up of several components. The first one is the core of the box and this is the content. It might be text or image or video or something else. Than we have padding, surrounded by the border and the outer component is called margin.

Adding Style

Adding style to a html document can be done in different ways. We can link our HTML document to a CSS document using the link tag in the head element. If we want to put two boxes next to each other we have to do some positioning. To do so we're using the flex box layout. In order to achieve our goal we have to apply this style to the parent container. Than all child containers will align in a row.

6.Python

Python expressions

Just like the grammar rules in natural languages there are rules in every programming language called syntax that have to be followed. Expression is something that has a value. We can also make an expression by combining other expressions with a set of operators (e.g. +, -, *, /).

Variables

Variable is a name created by us that refers to a value. The way to introduce a variable is by using an assignment statement (e.g. name = expression). A name could be any combination of characters as long as it starts with a letter and is not one of the Python keywords. Variables can change their value.

Strings

String is a sequence of characters surrounded by quotes. Quotes can be single, double or in case of multiple lines triple. Valid strings start and end with the same quotes. We can join (concatenate) strings using + operator but we can't combine strings with numbers.

Another thing we can do with strings is to extract sub sequences or single characters. We can extract character using square brackets and an expression (e.g. [expression]). To select a sub sequence we need to set a start and end position separated by a colon inside the square brackets (e.g.[expression : expression]). All expressions must be numbers.

There are different operations or methods that can be applied on strings. This is one of them:

  • find
    • Using one parameter
      Use: string1.find(string2)
      Explanation: finds where string2 occurs in string1.
      Return value: position of the first occurrence – a number. If none -1.
    • Using two parameters
      Use: string1.find(string2, number)
      Explanation: finds where string2 occurs in string1 after the first occurrence if number is greater that the return value of the first position.
      Return value: position of the second occurrence – a number. If none -1.
Procedures/ Functions

Function is a callable sub-program within a larger program. To create a function we need to give it a name followed by opening parentheses, list of inputs and closing parentheses followed by a colon(e.g. function(input1, input2): ). Inputs are called operands or arguments. The line after the name and the input of the function is the body. Inside the body is the place where we write/ give all instructions for this function to perform usually ending with a return statement. The return statement is the function’s output. To summarise, this is how a function work:

INPUT >>> FUNCTION (SOME CALCULATIONS/ OPERATIONS) >>> OUTPUT

One of the main rules in programming is to avoid repetition. Functionas are extremely helpful in this regard. If we have a set of operations that we need to perform often, we can build a function and call it every time we have to apply those operations. We can also use functions within functions, meaning we can break a complicated and long set of operations into a number of different simpler functions. And last but not least, in every programming language there is an extensive list of built in functions we can use whenever needed(e.g. append(), index(), len()).

Making decisions

To make code behave differently we need to make decision based on what the data is. To make those decisions we need some comparisons. Python provides different operators for comparison.

This is a list with some of them:

  • < less then
  • > greater then
  • <= less or equal
  • >= greater or equal
  • == equal
  • != not equal

All of those operate on number, but the output value has a Boolean value of True or False.

If Statements

Having the knowledge to make decisions, we now need something to make code behave different. The way to do that is by using if statement. The syntax of if statement is:

if <test expression>:

<first block of code>

else:

<second block of code>

If the test expression is true, the first block of code is executed – if not the second block is executed.

While loop

While loop is a construct that executes code over and over again until some condition is met.

While syntax:

while <test expression>:

<block of code>

Until the test expression is true, the block of code inside the loop will be executed over and over again. There is a way of stopping the loop before the test expression is false and it is done by using the break statement. This could be very handy if we have another condition inside the loop and it’s fulfilled before the test expression changes its value.

Debugging

Debugging is the process of locating and fixing or bypassing bugs (errors) in computer program code. To debug a program is to start with a problem, isolate the source of the problem, and then fix it.

Important steps in debugging a problem:

  • Examine error messages and google for solution.
  • If your code doesn't work, find an example code, run it and see how it works.
  • When working with example code, always make sure it's working by running it first.
  • If there is no error message but your code doesn't do what it's supposed to do – use print statements to check value for different variables.
  • keep copy of your code before making any changes
Structured data: Lists

String is a type of structured data but its use is limited to characters only. To store different kinds of data we need another structured data type called list. Lists can store sequences of anything. Just like in string we can access different elements of a list using the same syntax: random_list[0] gives us access to the first element of random_list. We can slice sub sequences: random_list[0 : 3], this will give us all elements between index 0 and 3 (not incl.). As mentioned before lists can store all kind of data including other lists. A list within a list is called nested list.

There is another big difference between strings and lists and that’s the list’s ability to support mutation. Mutation means we can change the value of a list after it’s been created.

List operations:

  • append()
    • Use: list.append(element)
      Explanation: append adds a new element at the end of the list. This method is mutator.
  • +
    • Use: [0, 1] + [2, 3]
      Explanation: Joins two lists together: [0, 1, 2, 3].
  • len()
    • Use: len(list)
      Explanation: gives us the number of elements in the list. In case of nested lists, len only counts the number of the outer elements.
  • index()
    • Use: list.index(value)
      Explanation: index looks for value inside the list and returns the value position, where it first occurs. If the value we are looking for isn’t in the list, index returns error.
  • in
    • Use: value in list
      Explanation: looks for given value in the list. If found, returns True, if not False.
Aliasing

When we have two different ways to refer to an object, that’s called aliasing.

For loop

For iteration inside lists, Python provides another construct called For loop.

For - syntax:

for name in list:

<block of code>

Name is a variable. We assign the first element to this variable and do something in the block with the code. Once we finish with this element, the for loop assigns next element to name and so on until we go through all the elements inside the list.

7.Object Oriented Programing (OOP)

What is OOP?

As the name suggest Object Oriented Programming is a type of language based on the concept of objects. This type of programming defines not only the data type of a structure, but also the types of operations (methods) that can be applied to the data structure. These data structures are called objects and they contain data in the form of fields, aka attributes.

OOP is a very powerful tool when it comes to large scale projects. Here are some of the advantages:

  • Modules created in a OO program can easily be reused in other programs. Once a module is created, knowledge of how it actually works isn't neccessary for its use. In procedure-based programming, coders needed understand the details of a piece of code before using it. Objects in object-based program development can be extended to include new attributes and behaviors and they can be reused within and accross applications. Beacause of this OOP proves to be more reliable software development tool, suitable for large scale projects.
  • It's easy to maintain and modify existing code. Because of its modular nature, updates can be applied to individual modules without the need of large-scale changes. If there are issues, it's easier to locate the problem since we apply the changes gradually.
  • Reuse of code helps for faster development. The extensive list of libraries in and outside the standart library helps us build our application and saves time. The code we write could be reused in other projects to come, thus saving us time and money.
  • All of the above helps programmers spend less time writing code and allows them to analize and verify the software, resulting in higher quality and fault proof applications.

What is a Class?

Class is code in a file in or outside Python Standart Library. In that class we have function called __init__ (short for initialize). Every time we call a class we run this method by default and it creates space in memory for a new object. The newly created object can access the rest of the methods inside the class. Every class has a set of attributes (properties) that apply to multiple objects created of that class. We can think of a class as a blue print and we can think of its objects as examples of objects of that blue print.

Importing Classes

Built in classes:

Part of the library,no need to import.

Outside the library:

The from... import statement in Python lets us import specific attributes from a module into the current namespace. In our example we have from twilio.rest import TwilioRestClient When we look in twilio github page we can find a directory called twilio. Inside is another directory called 'rest'. In rest, there is a python file where we can find class called TwilioRestClient. Basically what the from...import statement does is, finding class TwilioRestClient in /twilio/rest/__init__.py and implementing this code in our file. We can also import directories but in this case when we create object we have to write the name of the directory in front of the class name followed by a dot.

Note:

  • Library is a generic term for a package or a single module being published.
  • Python Standart Library is a wide range of built-in modules.
  • Python Package Index is a collection of modules, packages or even entire application development frameworks.
  • Package is a collection of modules.
  • Module is a file containig code for a Class.
Creating Classes

First thing when creating a class is to write the keyword class and a Name. It's recommended the name we give to have logical connection with the class and to start with a capital letter. A class can have both data and methods(functions inside a class). As mentioned prior a class can have multiple objects or instances. When we create an object of a class, the class's constructor is called. This is the __init__ method inside the class. This is where we initialize all the data associated with the object/ instance. The constructor uses the keyword self. Self refers to the object/ instance. All variables associated with a specific instance/ object are called instance variables. Those are unique to an object and can be accessed using the 'self' keyword inside a class and the instance name outside the class. All functions inside a class associated with an object/ instance and have self as an argument are called instance methods.

Class variables

Class variables are defined in the class, outside class's methods. Class variables can be accessed by every instance (object) of that class. If a class variable is not expected to change it's value (i.e. constant), Python conventions recommend use of capital letters for the name, e.g. CLASS_VARIABLE.

Predefined class attributes

Classes have five predefind attributes. Here are some of them:

  • __name__ Class name
  • __doc__ This atribute is defind right after the name of the class. It's wrapped in triple quotes and represents class's documentation
  • __module__ When called this module gives us the name of the file containing our class
Inheritance

When we have a class with a number of atributes and another class with the same attributes plus some more, instead of rewriting everything again for the new class we can use what we already have and just add the new attributes. We can do this by declaring our new class as a child of the first one. Our child class can now access all the attributes declared in its parent class(the first class).
Example:

This relationship between classes is called Inheritance. Another feature of inheritance is method reuse. This means we can access and use methods in our Parent class with an object of a Child class.

When a method defined in the child class overrides a method with the same name in the parent class, it's called method overriding.

A good example for inheritance can be made with what we've already learned about styling HTML pages. For example, if we set certain properties for the body element like font-style, color or family, all these properties will be inherited by the child elements: paragraphs, lists etc. But if we set different properties for one of the elements, this element will appear different than the rest inside the body. The reason for this is just like in method overriding in classes, we override the inherited values with the local inside the element.
